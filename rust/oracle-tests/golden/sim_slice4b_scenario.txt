# Control bits (7-bit ControlState, see worm.hpp Control enum):
#   Up=1  Down=2  Left=4  Right=8  Fire=16  Change=32  Jump=64
#
# Aiming convention (cossin_table indexed 0..127): 0=up, 32=right, 64=down,
# 96=left. worm0's seed-42 resting aim points roughly horizontal-into-the-floor, so a
# shot fired flat explodes on contact INSIDE the solid dirt (no Background in the
# window => no dig). To DIG we must ARC the shot: hold Up(1) to raise the gun, Fire,
# and let gravity parabola the ball back DOWN onto the dirt SURFACE, where the 16x16
# explosion window straddles sky (Background) above and dirt below. ProcessAiming
# accelerates slowly (AimAcc 4000), so ~25 held Up ticks are needed for a usable arc.
#
# THE GREENBALL (weapons/greenball.cfg): gravity=700 (parabola), speed=200,
# timeToExplo=0 (NO timeout — it explodes ONLY on a ground/worm hit, expl_ground=true),
# dirt_effect=6, create_on_exp=-1, splinter_amount=0. Its dirt texture (6) is ADDITIVE
# (ndrawback=false): draw_dirt_effect CREATES dirt, and it can only do so in
# Background cells. THIS IS THE WHOLE POINT OF 4b: if the explosion window is all solid
# dirt (a buried/flat shot) or all open sky (a shot that never lands), the `level` hash
# NEVER moves. The window must STRADDLE the surface. Because create_on_exp=-1 and
# splinter_amount=0, greenball spawns NO secondary objects: `sobjects`/`nobjects` stay
# empty for the whole run (secondary-object spawning is 4c, a different weapon).
# `weapon 0 GREENBALL` puts it in slot 0; the dumper hardcodes current_weapon=0.
#
# HARD CONSTRAINTS (the implementer verified every input line below obeys these):
#   * NEVER set Left(4) and Right(8) together on the same worm/tick. worm0 only ever
#     sets Up / Down / Fire, so it never walks and never digs via the L+R dig path.
#   * worm1 starts visible 0 -> Worm::Process is gated `if (visible)` (worm.cpp:218),
#     so worm1 NEVER moves, fires, or draws RNG: its component hash is CONSTANT, the
#     (omitted) worm-hit loop can never apply, and greenball's worm_collide=true /
#     detect_distance=3 can never trigger against it. The no-shot-within-
#     detect_distance-of-a-VISIBLE-worm rule is therefore trivially satisfied.
#   * worm0 stays health 100, in-bounds, grounded.
#   * The IMPACT MUST STRADDLE THE SURFACE (Background cells present in the 16x16
#     window) or `level` never moves. Both shots are arced for exactly that; the golden
#     proves it (level: pristine -> moves at the shot-1 explode tick -> moves AGAIN at
#     the shot-2 explode tick, two distinct dug values).
